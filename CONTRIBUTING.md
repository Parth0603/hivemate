# Contributing to SocialHive

Thank you for your interest in contributing to SocialHive! This document provides guidelines and instructions for contributing.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/socialhive-platform.git`
3. Create a feature branch: `git checkout -b feature/your-feature-name`
4. Follow the setup instructions in [QUICK_START.md](QUICK_START.md)

## Development Workflow

1. Make your changes in your feature branch
2. Write or update tests for your changes
3. Run tests: `npm test`
4. Run linting: `npm run lint:fix`
5. Commit your changes with clear, descriptive messages
6. Push to your fork
7. Open a Pull Request

## Code Style

- Use TypeScript for all new code
- Follow the existing code style (enforced by ESLint)
- Write meaningful variable and function names
- Add comments for complex logic
- Keep functions small and focused

## Testing

- Write unit tests for new functions
- Add integration tests for new features
- Ensure all tests pass before submitting PR
- Aim for good test coverage

## Commit Messages

Use clear, descriptive commit messages:
- `feat: add user profile photo upload`
- `fix: resolve connection request bug`
- `docs: update API documentation`
- `test: add tests for messaging service`

## Pull Request Process

1. Update README.md if you've added features
2. Ensure all tests pass
3. Update documentation as needed
4. Request review from maintainers
5. Address any feedback

## Questions?

Open an issue for questions or discussions about contributions.
